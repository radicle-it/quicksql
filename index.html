            Tabelle
            

                

                    
nome_tabellaCrea tabella con PK automatica

                    
nome /insert NGenera N righe di dati campione

                    
nome /restAbilita ORDS REST sull'oggetto

                    
nome /auditAggiunge colonne created/updated/by

                    
nome /rowversionAggiunge row_version per OCC

                    
nome /apiGenera Table API (TAPI package)

                    
nome /immutableTabella immutabile (no UPDATE/DELETE)

                    
nome /historyTemporal history tracking

                    
nome /sodaSODA document store

                

                
Le tabelle indentate sotto un'altra diventano automaticamente figlie con FK verso il padre.

            Settings
            

                
Inserisci a fine file oppure usa il pannello ⚙ Settings nell'header.

                # settings = { pk: seq, semantics: CHAR, db: 23c }
                

                    
pkseq | guid | identityDataType | none

                    
semanticsCHAR | BYTE

                    
dbVersione Oracle: 23c, 19c, 21c …

                    
prefixPrefisso tabelle (es. app_)

                    
schemaNome schema Oracle (es. HR)

                    
editionableyes | no

                    
auditcolsyes | no

                    
rowversionyes | no

                    
DVtrue | false — abilita Duality View

                    
languageLocale per nomi campione (es. it)

                

            

        


        

            Esempi rapidi
            

                
Schema con gerarchia automatica:

                departments /insert 2
   name /nn
   employees /insert 5
      name /nn vc50
      email /lower
      salary num(10,2)
                
Tabella con trigger e JSON:

                products /audit
   name vc100 /nn /upper
   price num(10,2) /nn
   specs json
                
FK esplicita + audit:

                orders /audit /insert 10
   customer_id /fk customers
   total num(10,2) /nn
   status /check OPEN,CLOSED,PENDING
            

        


        

            Scorciatoie tastiera
            

                

                    
TabIndenta di 3 spazi

                    
EscChiude Help e Settings

                

                
Il DDL si aggiorna in tempo reale a ogni keystroke. Il diagramma ER si aggiorna 350 ms dopo l'ultimo tasto.

                
Il testo QSQL viene salvato automaticamente in localStorage e ripristinato al prossimo accesso. Il pulsante Condividi link codifica schema + posizioni ERD nell'URL.
