        QuickSQL Reference
        
    

    


        

            Tables
            

                

                    
table_nameCreates a table with auto primary key

                    
name /insert NGenerate N sample data rows

                    
name /restEnable ORDS REST on the object

                    
name /auditAdd created/updated/by columns

                    
name /rowversionAdd row_version column for OCC

                    
name /apiGenerate Table API (TAPI package)

                    
name /immutableImmutable table (no UPDATE/DELETE)

                    
name /historyTemporal history tracking

                    
name /sodaSODA document store

                

                
Tables indented under another table automatically become children with a FK to the parent.

            Settings
            

                
Add at the end of the file or use the ⚙ Settings panel in the header.

                # settings = { pk: seq, semantics: CHAR, db: 23c }
                

                    
pkidentityDataType | guid | seq | none

                    
genpkyes | no — auto-generate PK column

                    
prefixpkwithtnameyes | no — prefix PK col with table name

                    
prefixObject prefix (e.g. app_)

                    
schemaOracle schema name (e.g. HR)

                    
dialectSQL dialect — oracle (default)

                    
dbOracle version: 23c, 19c, 21c …

                    
semanticsCHAR | BYTE

                    
dateDATE | TIMESTAMP | tswtz | tswltz

                    
booleannative (23c) | yn (char Y/N)

                    
languageSample data locale: EN | JP | KO

                    
auditcolsyes | no — created/updated/by columns

                    
rowversionyes | no — row version for OCC

                    
rowkeyyes | no — alphanumeric row identifier

                    
aienrichmentyes | no — AI enrichment columns

                    
dropyes | no — include DROP statements

                    
insertsyes | no — generate INSERT data

                    
dvyes | no — enable JSON Duality Views

                    
editionableyes | no

apexyes | no — APEX integration

            Quick Examples
            

                
Schema with automatic parent-child hierarchy:

                departments /insert 2
   name /nn
   employees /insert 5
      name /nn vc50
      email /lower
      salary num(10,2)
                
Table with trigger and JSON column:

                products /audit
   name vc100 /nn /upper
   price num(10,2) /nn
   specs json
                
Explicit FK + audit columns:

                orders /audit /insert 10
   customer_id /fk customers
   total num(10,2) /nn
   status /check OPEN,CLOSED,PENDING
